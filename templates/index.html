pipeline

How a verdict gets made

01 / extract

Parse the URL

The scheme, host, path and query are pulled apart and turned into 20+ numeric signals — length, digit ratio, subdomain count, IP-as-host, suspicious keywords, free-TLD usage, and more. No DNS lookups, no page is fetched.

02 / analyze

Run heuristic checks

Each signal is evaluated against known phishing patterns from security research — the same checks a human analyst would eyeball, just run instantly and shown in the scan log above.

03 / classify

Score with a trained model

A Random Forest classifier, trained on the feature set, turns the signals into a single probability of phishing — the risk score you see in the gauge.
